# TD-MPC2: Scalable, Robust World Models for Continuous Control

> **作者**：Nicklas Hansen, Hao Su, Xiaolong Wang（三位均为同等贡献；Su 和 Wang 为共同通讯作者）
> **机构**：University of California San Diego (UCSD)
> **年份/会议**：2024 / ICLR 2024
> **arXiv**：arXiv:2310.16828

## 一句话总结

TD-MPC2 通过一系列算法改进（SimNorm、离散回归、Q 函数集成、最大熵策略先验、可学习任务嵌入）使 TD-MPC 从单任务扩展至 104 个任务的统一超参数框架，并首次证明隐式世界模型的性能随参数规模（1M → 317M）单调提升，实现了跨域、跨本体、跨动作空间的大规模多任务连续控制。

## 核心贡献

- **单一超参数集覆盖 104 个任务**：无需任何任务级调参，在 DMControl（39 任务）、Meta-World（50 任务）、ManiSkill2（5 任务）、MyoSuite（10 任务）上全面超越 SAC、DreamerV3 和原始 TD-MPC。
- **可扩展的隐式世界模型**：首次证明 TD-MPC2 的性能随模型规模单调提升（1M→317M 参数），而原始 TD-MPC 在规模增大时性能反而下降；317M 参数的单一 agent 能在 80 个多样化任务上同时运行。
- **关键算法创新**：SimNorm（Simplicial Normalization）稳定潜空间表征、离散回归（101 个 bin）使损失量级与奖励尺度无关、5 个 Q 函数集成 + 1% Dropout、最大熵策略先验（SAC 风格）、可学习任务嵌入（ℓ₂ 归一化）。
- **开源生态**：公开 300+ 模型检查点、545M 转移数据集、训练/评估代码（https://tdmpc2.com）。

## 方法

### Enc()

输入为低维状态向量（或视觉 RL 中的 64×64 图像）。编码器 h 将观测 s 和任务嵌入 e 映射到潜在状态 z：

```
z = h(s, e)
```

**状态输入架构**（5M 基础模型）：
```
Linear(S+T, 256) → NormedLinear(256, 256, Mish) → NormedLinear(256, 512, SimNorm)
参数量：167,936
```

**视觉输入**：将 MLP 替换为 4 层浅层 CNN（64×64 输入，random shift 数据增强）。

编码器使用**较低学习率**（1×10⁻⁴，其他组件为 3×10⁻⁴），输出维度 512，经 SimNorm 归一化。

**SimNorm（Simplicial Normalization）**：将潜在向量 z 分为 L 组（每组 V=8 维），每组经 softmax 归一化（各组之和为 1）：

```python
def simnorm(z, V=8):
    z = z.view(*z.shape[:-1], -1, V)
    z = softmax(z, dim=-1)
    return z.view(*z.shape[:-2], -1)
```

SimNorm 是 VQ-VAE 向量-类别的"软"版本（τ=1 softmax；τ→∞ 退化为 one-hot，τ=0 退化为均匀分布），天然偏向稀疏表示，是训练稳定性的关键，解决了原始 TD-MPC 的梯度爆炸问题。

### Pred()

潜在动力学预测器 d 在潜空间中做 action-conditioned 多步预测：

```
z' = d(z, a, e)
```

**架构**（3 层 MLP）：
```
NormedLinear(512+A+T, 512, Mish) → NormedLinear(512, 512, Mish) → NormedLinear(512, 512, SimNorm)
参数量：843,264
```

输入为 (z, a, e) 拼接（512 + 动作维 + 任务嵌入维），输出经 SimNorm 归一化的下一潜在状态 z'。

**递归展开**：预测链式进行 H 步（H=3），无需解码回观测空间。规划时使用 MPPI 在此模型上展开候选轨迹。

**联合嵌入预测损失（BYOL 风格）**：

```
L_dyn = ||z'_t - sg(h(s'_t))||²₂    （系数 20，stop-gradient 作用于目标编码器）
```

这是隐式世界模型的核心：预测器被训练为匹配编码器对真实下一观测的表征，而非重建像素。

### Dec()

**TD-MPC2 是无解码器（decoder-free）的隐式世界模型**，不重建任何原始观测。

- **训练时**：无解码器，仅通过联合嵌入预测损失（L_dyn）、奖励预测损失（L_r）和价值预测损失（L_q）共同训练。
- **规划时**：完全在潜空间中计算，不生成任何图像。

这与 Dreamer 系列（依赖像素重建）的根本区别：TD-MPC2 认为解码器会浪费表征容量在任务无关的视觉细节上。

### Reward / Value 建模

**奖励模型 R**：

```
r̂ = R(z, a, e)   →   输出 101 个 logit（离散 bin）
```

架构：3 层 MLP，输出 101 维 logit，使用**离散回归（Discrete Regression）**：

- 将奖励映射到 log 变换空间的 101 个均匀 bin
- 用交叉熵损失（soft target）而非 MSE 训练
- 关键优势：损失量级与任务奖励的绝对尺度无关，使单一超参数集成为可能

**Q 函数集成（Value Model）**：

- 默认 **5 个 Q 函数**（317M 模型为 8 个），每个均为 3 层 MLP + 1% Dropout
- 每个 Q 函数输出 101 个 logit（同样使用离散回归）
- TD 目标：随机子采样 2 个目标 Q 函数（Q̄，EMA 动量 0.99）取最小值（RED-Q 风格）

**策略先验 p（Policy Prior / Actor）**：

```
â = p(z, e)   →   输出 2A 维（均值 + log 标准差，最大熵 RL，SAC 风格）
```

策略先验用于 MPPI 规划时初始化部分候选样本（24/512），同时作为独立的 actor 训练。

**联合模型目标**（Eq. 3，时序衰减系数 λ=0.5）：

```
L(θ) = Σ_{t=0}^{H} λ^t [ 20·||z'_t - sg(h(s'_t))||²₂  +  0.1·CE(r̂_t, r_t)  +  0.1·CE(q̂_t, q_t) ]
```

**策略目标**（最大熵，Eq. 4）：

```
L_p(θ) = Σ_{t=0}^{H} λ^t [ α·Q(z_t, p(z_t)) - β·H(p(·|z_t)) ]
```

其中 α 通过移动统计（5th-95th 百分位归一化）自适应调整。

### 关键设计亮点

1. **SimNorm + 离散回归的双重稳定化**：SimNorm 解决了原始 TD-MPC 因无归一化导致的梯度爆炸，离散回归解决了不同任务奖励尺度不一致导致的超参数敏感性。两者共同使单一超参数集在 104 个任务上成为可能。

2. **MPPI 潜空间规划（MPC 框架）**：在每个决策步运行 MPPI（Model Predictive Path Integral），规划视野 H=3，迭代 6 次，候选 512 条，精英 64 条；部分候选（24 条）由策略先验初始化。完整 RL 回报估计：

   ```
   γ^H · Q(z_{t+H}, a_{t+H}) + Σ_{h=t}^{H-1} γ^h · R(z_h, a_h)
   ```

   相比 PlaNet/Dreamer 的 CEM，MPPI 通过软加权（temperature=0.5）而非硬截断精英来更新分布。

3. **可学习任务嵌入 + 动作掩码（多任务泛化）**：96 维可学习任务嵌入 e（ℓ₂ 范数 ≤1 约束），所有 5 个组件均以 e 为条件；动作维度补零至最大值并掩码无效维度，实现跨动作空间泛化。T-SNE 可视化显示语义相似任务自然聚类。

## 实验结果

**基准**：104 个任务（DMControl 39 + Meta-World 50 + ManiSkill2 5 + MyoSuite 10）。

**单任务性能（5M 参数，3 个随机种子，95% 置信区间）**：

- **DMControl（39 任务）**：TD-MPC2 > DreamerV3 > TD-MPC > SAC；原始 TD-MPC 在 Walker Stand/Walk 等任务出现梯度爆炸发散，TD-MPC2 全程稳定。
- **Meta-World（50 任务）**：TD-MPC2 在简单任务与基线相当，在难任务（Pick Place Wall、Shelf Place）上显著领先；DreamerV3 常无法收敛。
- **ManiSkill2（5 任务）**：Pick YCB（74 种 YCB 物体，最难）在 14M 步时 TD-MPC2 达到 **>60% 成功率**，其他所有方法（SAC、DreamerV3、TD-MPC）均接近 0%。
- **MyoSuite（10 任务）**：TD-MPC2 在大多数任务上达到可比或更好的性能（Key Turn Hard 上 TD-MPC 略早收敛）；MyoSuite 实验在报告前从未运行过，验证了超参数的零样本泛化性。

**大规模多任务扩展（80 任务，DMControl 30 + Meta-World 50）**：

| 参数量 | GPU 天数（RTX 3090） | 归一化分数 |
|--------|---------------------|-----------|
| 1M | 3.7 | 16.0 |
| 5M | 4.2 | 49.5 |
| 19M | 5.3 | 57.1 |
| 48M | 12 | 68.0 |
| 317M | 33 | 70.6 |

性能随 log(参数量) **近似线性增长**，317M 模型尚未饱和。相比之下，原始 TD-MPC 随规模增大性能下降。

**少样本微调（Few-Shot Finetuning）**：19M 预训练 agent 在 10 个保留任务上用 20k 步微调，归一化分数 47.0 vs 从头训练 24.0（约 2× 提升）。

**消融实验**（3 个最难任务 + 80 任务多任务）：

| 变体 | 分数 |
|------|------|
| 无规划（仅策略） | 42.2 |
| 无策略先验（仅规划） | 53.7 |
| 规划 + 策略先验（默认） | 54.2 |
| 无归一化 | 46.8 |
| 仅 SimNorm | 51.0 |
| LayerNorm + SimNorm（默认） | 54.2 |
| 连续回归 | 49.6 |
| 离散回归（默认） | 54.2 |
| 2 个 Q 函数 | 53.5 |
| 5 个 Q 函数（默认） | 54.2 |
| 任务嵌入无归一化 | 46.6 |

## 局限性

- **不支持离散动作空间**：MPPI 专为连续动作设计，无法直接用于 Atari 等离散动作环境；作者建议用 MCTS 替换 MPPI（类似 MuZero）作为未来方向。
- **依赖奖励信号**：TD-MPC2 需要任务奖励驱动学习，无奖励的自监督预训练尚未探索。
- **离线 RL 局限**：当前的保守性离线 RL 技术不适合无缝的离线预训练 → 在线微调过渡；论文提出的测试时正则化（test-time regularization）是变通方案但仍需调整系数 c。
- **性能未饱和**：317M 参数模型尚未接近饱和，实现真正通用行为可能需要比现有多出几个数量级的任务数量。
- **视觉 RL 分辨率有限**：仅在 64×64 分辨率下测试，高分辨率视觉 RL 未深入探索。
- **奖励误设风险**：奖励函数设计不当可能导致难以预料的行为；物理机器人控制中无约束自主性可能导致灾难性失败。

## 与同范式演进关系

| 论文 | 与 TD-MPC2 的关系 |
|------|-----------------|
| **PlaNet (2019)** | 同样在潜空间规划（CEM），但需要像素重建解码器；TD-MPC2 是无解码器的隐式世界模型 |
| **Dreamer (2020)** | 生成式世界模型（RSSM + 像素重建），在潜空间做 actor-critic；TD-MPC2 无解码器，用 MPPI 规划 + 价值引导 |
| **DreamerV3 (2023)** | 主要对比基线，同样追求跨任务统一超参数；DreamerV3 是生成式（重建）模型，在操控任务和高维运动任务上被 TD-MPC2 显著超越 |
| **TD-MPC (Hansen et al., 2022)** | 直接前身；TD-MPC2 的核心改进：SimNorm 稳定训练、离散回归消除尺度敏感性、Q 函数集成 + Dropout、最大熵策略先验、可学习任务嵌入、动作掩码；原始 TD-MPC 规模增大性能下降，TD-MPC2 单调提升 |
| **MuZero (2020)** | 同样是隐式（无解码器）世界模型，使用 MCTS 规划；TD-MPC2 用 MPPI（连续动作），未来可用 MCTS 扩展到离散动作 |
| **SAC (2018)** | TD-MPC2 的策略先验借鉴 SAC 的最大熵 RL 目标；SAC 是主要的无模型基线，TD-MPC2 在所有任务上均优于 SAC |

**TD-MPC vs TD-MPC2 核心差异汇总**：

| 方面 | TD-MPC | TD-MPC2 |
|------|--------|---------|
| 潜空间归一化 | 无（→ 梯度爆炸） | SimNorm |
| 奖励/价值回归 | 连续 MSE | 离散交叉熵（101 bin） |
| Q 函数数量 | 2 个，无 Dropout | 5 个（默认），1% Dropout |
| 策略先验 | 确定性 + 高斯噪声 | 最大熵（SAC 风格） |
| 超参数 | 需要任务级调参 | 单一集合覆盖 104 任务 |
| 多任务 | 受限（共享观测/动作空间） | 跨域/跨本体/跨动作空间 |
| 规模扩展 | 规模增大→性能下降 | 单调提升（1M→317M） |
